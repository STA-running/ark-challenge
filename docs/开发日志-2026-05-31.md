### 2.3 种族数据提取

**发现**：PRTS Cargo API 的 chara 表中**没有 race 字段**，种族信息在干员页面 wikitext 的 "人员档案set" 模板中。

**方案**：使用 MediaWiki `action=query&prop=revisions&rvprop=content` API 批量获取 wikitext → 正则 `|种族=xxx` 提取。

**效率**：9 次 API 调用（8次×50人 + 1次×17人），覆盖 417 名干员，耗时约 20 秒。

**结果**：409人有种族信息（98.1%），8人无种族（全为机器人），44 种族种类。

**产出文件**：
- `docs/种族数据提取总结.md` — 完整经验总结
- `docs/operator_races.json` — 种族数据
- `scripts/extract_races_api.py` — 可复用提取脚本

## 三、关键设计决策

| # | 决策 | 理由 |
|---|------|------|
| 1 | 种族用中文名存储（"萨卡兹"） | PRTS 直接返回中文，无映射成本 |
| 2 | race 与 nation/org 分三个独立维度 | 语义不同（种族≠国籍≠组织），不可互相替代 |
| 3 | race 数据单独维护，构建时合并 | Cargo API 无 race 字段，需独立提取 |
| 4 | rareRaceOnly 用人数阈值而非具体种族 | 动态适配，游戏更新后自动生效 |
| 5 | 缺失数据全部用 null | JSON 原生类型，前端判断简洁 |
