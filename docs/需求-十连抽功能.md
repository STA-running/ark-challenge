# 十连抽取功能 · 需求分析文档

## 概述

在编队构建阶段，提供「十连抽取」替代手动选干员，让随机性融入自限挑战流程。

---

## 1. 核心规则

### 1.1 抽取规则

| 项 | 值 |
|------|------|
| 每次抽取人数 | **10 人** |
| 抽取方式 | **不放回**（同一轮内不重复） |
| 可锁定轮数 | **3 次** |
| 总锁定干员上限 | 30 人 |
| 最终编队来源 | **仅限**已锁定干员 |
| 标签约束 | 全程受当前已确认标签约束 |

### 1.2 选定规则

- 每次十连后，用户可点击 **「锁定这轮结果」** 或 **「不要」**
- **「不要」每轮最多用 10 次**，用完后必须锁定或十连结束
- 「不要」的计次在**锁定后重置**（下一轮又有 10 次）
- 锁定后该 10 人从合法池中**永久移除**，后续十连不再出现
- 选满 3 轮后十连功能关闭

### 1.5 十连概率规则

**第 1~9 抽** — 常规加权：
| 星级 | 6★ | 5★ | 4★ | 3★ |
|------|----|----|----|----|
| 概率 | 10% | 20% | 60% | 10% |

**第 10 抽** — 先判 50% 出 6★，未命中则走常规加权：

| 星级 | 概率规则 | 有效概率（表达式 → 值） |
|------|---------|---------------------|
| 6★ | 先判 50% + 剩余 50%×10% | 50% + (50%×10%) = **55%** |
| 5★ | 剩余 50%×20% | 50%×20% = **10%** |
| 4★ | 剩余 50%×60% | 50%×60% = **30%** |
| 3★ | 剩余 50%×10% | 50%×10% = **5%** |

- 50% 判定仅合法池有 6★ 时生效，否则走常规加权
- 6★ 上不封顶，不放回

### 1.3 模式互斥

- **十连模式与手动模式互斥**
- 选中「十连抽取」后，干员选择视图切换为十连模式
- 编队时只从已锁定的 3 批干员中选择
- 无法切回手动选干员，反之亦然

### 1.4 锁定池的唯一性

```
此局挑战的编队来源 = 已锁定的 3 批干员（最多 30 人）
                 ≠ 全部干员
```

## 3. 完整流程

```
关卡+标签确认
    ↓
编队阶段
    ↓
┌──────────选择编队方式──────────┐
│ [✋ 手动选择] [🎰 十连抽取]      │
└──────────┬───────────────────┘
           ↓ 选择十连 → 进入十连流程
           │
    ┌──────────────────────────┐
    │ 第 N 轮十连 (N=1,2,3)     │
    │ 从合法池抽 10 人          │
    │ 展示结果                  │
    │                          │
    │ [🔄 不要 还剩X次] [✅ 锁定] │
    └────────┬─────────────────┘
             │
   ┌─────────┴─────────┐
   │ 点了「锁定」        │  │ 点了「不要」(X>0)
   │                    │  │
   │ 这10人加入锁定池    │  │ X-=1, 重新十连
   │ 从合法池移除这10人  │  │
   │ N+=1               │  │
   │                    │  │
   │ N < 3 → 继续下一轮  │  │
   │ N = 3 → 十连结束    │  └────────
   └────────────────────┘
           ↓ N=3
    ┌────────────────────┐
    │ 🎰 十连结束                     │
    │ 已锁定 3 批，共 X 名干员         │
    │ 只能从这 X 人中编队             │
    │ [📋 开始编队]                   │
    └────────────────────┘
             ↓
    ┌────────────────────┐
    │ 锁定池编队模式                   │
    │ 3批干员分栏展示                  │
    │ 点击/双击加入编队                │
    └────────────────────┘
```

## 5. 边界情况

| 场景 | 行为 |
|------|------|
| 合法池 < 10 人 | 有多少抽多少 |
| 合法池 = 0 | 按钮灰色禁用，提示「无可抽取干员」 |
| 合法池不足3轮抽满30人 | 提前结束，按实际已锁定轮次处理 |
| 「不要」次数用完(10次) | 「不要」按钮变灰，仅剩「锁定」可选 |
| 编队已满 12 人 | 选取操作自动禁用 |

---

## 6. 维护性设计原则

- **概率表**：代码内常量，后续可抽出为 JSON 配置
- **抽取次数**：10连/3轮/10次=可配置常量
- **锁定池结构**：数组+批次号，扩展方便（可增批次数）
- **UI 组件化**：十连操作 ≈ 独立子组件，与手动模式完全隔离
